# clinic-template-stoma

Reusable Next.js 16 (App Router, `output: 'export'`) template for a dental clinic landing page.
Palette C ŌĆö Soft hybrid: `#1E3A5F` / `#7BA3C0` / `#F4F1EC`.

## Quick-start pipeline

### 1. Clone and install

```bash
git clone <your-fork> clinic-my-name
cd clinic-my-name
pnpm install
```

### 2. Fill in clinic data

Edit **`content/clinic.ts`** ŌĆö replace every `// TODO` placeholder:
- `name`, `legalName`, `inn`, `license`
- `address` (street, district, lat/lng for map)
- `phones`, `email`, `website`, `yclientsUrl`
- `social.yandexMapsId` (numeric ID from Yandex Maps URL)
- `rating`, `metro`, `hours`, `features`, `promos`

Edit **`content/doctors.ts`** ŌĆö add real doctors, one object per doctor.
Photos go in `public/images/raw/doctors/<id>.jpg` (see step 4).

Edit **`content/faq.ts`** ŌĆö customise answers (address, brands, etc.).

### 3. Scrape and parse prices

Place the scraped price JSON at `../research/prices/all_prices.json`
(one level above the repo root), then:

```bash
pnpm scrape:prices
```

This writes `content/prices.ts` with `priceCategories` and `featuredPrices`.
Tune `FEATURED_SUBSTRINGS` and `CATEGORY_LABELS` in `scripts/parse-prices.ts`
to match your clinic's price list structure.

### 4. Scrape and parse reviews

Place the scraped Yandex reviews JSON at `../research/reviews_yandex.json`, then:

```bash
pnpm scrape:reviews
```

This writes `content/reviews.ts` with the top-12 five-star reviews.

### 5. Add images

Drop raw clinic photos into `public/images/raw/clinic/` (jpg/png).
Drop doctor headshots into `public/images/raw/doctors/` (jpg/png, filename = doctor `id`).

Then optimize:

```bash
pnpm images
```

This generates avif/webp/jpg at 640/1280/1920w into `public/images/`.

### 6. Update app URLs

In `app/layout.tsx` change `metadataBase` from `https://example.vercel.app` to your domain.
Same in `app/sitemap.ts` (`BASE` constant).
In `components/JsonLd.tsx` the `SITE_URL` constant.

### 7. Build and preview

```bash
pnpm build        # runs next build ŌåÆ out/
pnpm preview      # serves out/ on :3000
```

### 8. Run smoke tests

```bash
pnpm test:e2e
```

Requires the static `out/` to exist (run `pnpm build` first).

### 9. Deploy

Push `out/` to Vercel / Netlify (static hosting, no server needed).
